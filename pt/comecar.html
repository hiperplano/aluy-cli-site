do zero à primeira tarefa

Comece em minutos.

Quatro passos: instalar, configurar provider e chave, rodar os primeiros comandos e seguir em frente. Sem segredo.

1

Instalar

baixar & instalar
bash — install aluy
$ curl -fsSL https://aluy.dev/install.sh | bash
ou com o cmd tradicional
cmd — install aluy
C:\>
1 linha · sem sudo·instala em ~/.aluy·re-rodar atualiza
Linux · macOS · Windows Node ≥ 20 ~150 MB livres conexão no 1º run

Terminal recomendado: WezTerm — o terminal mais adequado para usar o Aluy CLI (melhor renderização dos glifos da TUI; ou rode com --ascii).

2

Configurar — provider, chave & modelo

No primeiro run, um assistente abre. Você escolhe o provider, cola a chave e seleciona o modelo — nessa ordem.

a · provider

Broker por padrão, ou um provider direto (com a sua credencial). Troque depois com /provider.

b · chave

A credencial vai direto pro cofre do SO — nunca em texto, nunca no repo. Você não digita de novo.

c · modelo

Escolha o tier com /model. Ajuste o esforço de raciocínio com /effort quando precisar.

validar · /doctor

Rode /doctor para confirmar modelo, chave, conexão e ferramentas — read-only, com dica de conserto.

3

Abra a sessão e experimente

$ aluy abre a sessão neste projeto
/doctor checa a saúde da instalação
/init prepara as deps e o AGENT.md do projeto
/cycle "rode os testes e conserte o que quebrar"
/memory list o que o agente já lembra aqui

/ abre o menu · Ctrl+P abre a paleta · /help lista tudo
