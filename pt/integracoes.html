ecossistema

Providers & integrações.

O Aluy CLI nunca fala com um provider de modelo direto com seus dados expostos — a chamada ao modelo vai pelo broker (ou pelo seu próprio provider) na borda. Aqui está tudo a que ele se conecta: os providers de modelos que roteia e os projetos abertos sobre os quais é construído.

providers de modelos

Para onde vai a chamada ao modelo.

OpenRouter O gateway padrão — roteamento multi-provider que o broker usa de fábrica. openrouter.ai → TokenRouter Gateway de roteamento alternativo (OpenAI-compatível), acessível pelo broker. tokenrouter.io → Anthropic Modelos Claude — provider direto (sua chave ou OAuth). anthropic.com → OpenAI Modelos GPT — provider direto (sua chave). openai.com → Google · Gemini Modelos Gemini, acessados pela API OpenAI-compatível. ai.google.dev → DeepSeek Modelos DeepSeek — provider direto, pareável com o tier Custom. deepseek.com →
projetos que integramos

Os projetos abertos sobre os quais é construído.

MCP Model Context Protocol — o padrão aberto para conectar ferramentas. modelcontextprotocol.io → Ollama Roda modelos localmente — sidecar opcional. ollama.com → mem0 Memória de longo prazo — sidecar opcional. mem0.ai → headroom Recuperação de context-headroom — sidecar opcional. github.com/headroomlabs-ai → WezTerm O terminal recomendado para o Aluy CLI. wezterm.org → Node.js O runtime sobre o qual o CLI roda (Node ≥ 20). nodejs.org → Ink A interface de terminal (React) por trás da TUI. github.com/vadimdemedes/ink → Codex Compatibilidade — lê arquivos AGENTS.md do projeto. github.com/openai/codex → Claude Code Compatibilidade — lê arquivos CLAUDE.md do projeto. anthropic.com/claude-code →
